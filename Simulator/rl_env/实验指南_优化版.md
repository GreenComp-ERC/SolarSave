# SolarChain RL 阶段一优化版实验指南

本文档用于继续优化阶段一 Leader Agent，并系统测试不同 seed、timesteps、PPO 超参数、奖励函数权重和安全约束。

所有命令默认在以下目录执行：

```powershell
conda activate SolarChain-rl
cd D:\Documents\SolarChain\SolarSave\Simulator
```

## 1. 优化版做了什么

优化版环境相比初版增加了：

- 多场景随机训练：`--scenario random`
- 增强状态：时间周期、短期供需预测、流动性趋势
- 动作平滑：`--action-smoothing`
- 单步动作限幅：`--max-action-delta`
- 更强回撤惩罚：`--drawdown-penalty`
- 回撤阈值惩罚：`--drawdown-threshold` 与 `--drawdown-threshold-penalty`
- 流动性安全线惩罚：`--liquidity-floor-penalty`
- 分场景 benchmark：输出每个冲击场景下 PPO 和静态 1:3 的对比
- 时间戳输出目录：每次实验单独保存，便于横向对比

输出目录中会保存：

```text
leader_agent.zip              # PPO 模型
train_args.json               # 训练参数
benchmark_args.json           # benchmark 参数
env_config.json               # 环境配置
benchmark_results.csv         # 明细结果
benchmark_summary.json        # 总体平均结果
benchmark_by_scenario.json    # 分场景结果
```

## 2. 推荐的 Level 2 OpenWeather 训练命令

这是当前建议的基础实验：

```powershell
python -m rl_env.train_ppo `
  --data-source openweather `
  --scenario random `
  --timesteps 300000 `
  --n-envs 4 `
  --seed 7 `
  --drawdown-penalty 2.0 `
  --oscillation-penalty 0.35 `
  --unmet-demand-penalty 0.50 `
  --liquidity-floor-penalty 0.35 `
  --drawdown-threshold 0.35 `
  --drawdown-threshold-penalty 2.0 `
  --action-smoothing 0.20 `
  --max-action-delta 0.08 `
  --learning-rate 0.0003 `
  --n-steps 1024 `
  --batch-size 256 `
  --gamma 0.98 `
  --gae-lambda 0.95 `
  --ent-coef 0.005 `
  --clip-range 0.2 `
  --output-dir rl_env\outputs\experiments `
  --run-name ow_l2_seed7_t300k `
  --timestamp-output
```

`--timestamp-output` 会生成类似目录：

```text
rl_env/outputs/experiments/20260509_173000_ow_l2_seed7_t300k/
```

## 3. 对应 benchmark 命令

将下面的 `<RUN_DIR>` 替换为训练输出目录。

注意：benchmark 的环境参数要和训练一致，尤其是 observation、动作平滑、惩罚系数，否则模型输入分布会不一致。

```powershell
$run = "rl_env\outputs\experiments\<RUN_DIR>"

python -m rl_env.benchmark `
  --model "$run\leader_agent.zip" `
  --data-source openweather `
  --episodes 20 `
  --seed 100 `
  --drawdown-penalty 2.0 `
  --oscillation-penalty 0.35 `
  --unmet-demand-penalty 0.50 `
  --liquidity-floor-penalty 0.35 `
  --drawdown-threshold 0.35 `
  --drawdown-threshold-penalty 2.0 `
  --action-smoothing 0.20 `
  --max-action-delta 0.08 `
  --output-dir "$run\benchmark" `
  --run-name eval_all `
  --timestamp-output
```

## 4. 快速 smoke test

用于确认代码能跑，不用于下结论：

```powershell
python -m rl_env.train_ppo `
  --data-source openweather `
  --scenario random `
  --timesteps 4096 `
  --n-envs 1 `
  --output-dir rl_env\outputs\smoke `
  --run-name smoke_train `
  --timestamp-output
```

然后 benchmark：

```powershell
$run = "rl_env\outputs\smoke\<RUN_DIR>"

python -m rl_env.benchmark `
  --model "$run\leader_agent.zip" `
  --data-source openweather `
  --episodes 1 `
  --output-dir "$run\benchmark" `
  --run-name smoke_eval `
  --timestamp-output
```

## 7. 回撤控制实验

如果 PPO 的流动性提升明显，但 `max_drawdown` 变差，优先测试：

```text
drawdown_penalty: 2.0, 3.0, 5.0
drawdown_threshold_penalty: 2.0, 4.0, 8.0
```

示例：

```powershell
python -m rl_env.train_ppo `
  --data-source openweather `
  --scenario random `
  --timesteps 300000 `
  --seed 7 `
  --drawdown-penalty 3.0 `
  --drawdown-threshold 0.30 `
  --drawdown-threshold-penalty 4.0 `
  --action-smoothing 0.20 `
  --max-action-delta 0.08 `
  --output-dir rl_env\outputs\experiments `
  --run-name ow_l2_dd3_threshold030 `
  --timestamp-output
```

## 8. 动作稳定性实验

如果 `mean_action_oscillation` 过高，测试：


## 10. 读结果时重点看什么

优先看：

```text
benchmark_summary.json
benchmark_by_scenario.json
```

关键指标：

```text
episode_volume              越高越好
mean_liquidity              越高越好
low_liquidity_steps         越低越好
total_unmet_demand          越低越好
max_drawdown                越低越好
mean_action_oscillation     越低越稳
```

不要只看总体平均，也要看分场景：

```text
normal
cloudy_supply_drop
factory_demand_spike
oversupply
demand_collapse
token_volatility
mixed_shock
```

一个适合进入阶段二的模型应该满足：

```text
1. mean_liquidity 明显优于 static_1_to_3
2. total_unmet_demand 明显低于 static_1_to_3
3. max_drawdown 不明显差于 static_1_to_3
4. mean_action_oscillation 不过高
5. 分场景没有明显灾难性退化
```

## 11. 常见注意点

- `openweather` 使用的是真实实时天气快照，不是完整历史天气数据。
- OpenWeather 结果会缓存到 `rl_env/cache/openweather_weather.json`。
- 如果想重新拉取天气，删除该缓存文件后重新运行。
- 训练和 benchmark 的环境参数要保持一致。
- 如果使用 `--legacy-observation` 训练，benchmark 也必须加 `--legacy-observation`。
- 输出目录带时间戳后，不同实验不会互相覆盖。
